# Contributing

Thank you for your interest in contributing to Advanced Multi Agent Orchestrator.

## How to contribute

1. Open an issue to describe the change.
2. Fork the repository.
3. Create a feature branch (`git checkout -b feat/my-change`).
4. Make your changes.
5. **Verify the quality of your documentation.**

   - If you use VS Code, install the `markdownlint` extension (by David Anson).
     It will highlight errors in real-time.
   - If you don't use VS Code, push your branch and let the CI validate it.
     Errors will appear in the "Checks" tab of your Pull Request.

   *No local linting toolchain installation is required to contribute.*

6. Open a pull request against `main`.

## Guidelines

- Keep the documentation clear and readable.
- Prefer small, focused changes over large sweeping edits.
- Update `CHANGELOG.md` when adding notable changes.
- Ensure CI passes before requesting a review.
